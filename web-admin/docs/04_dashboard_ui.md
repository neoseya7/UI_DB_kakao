# 🖥️ 04. Dashboard UI & Admin Operations

본 문서는 `src/app/(dashboard)` 내부의 UI 동작 명세서입니다.

## 1. 오늘의 대화 (Today's Chat - `/page.tsx`)
실시간으로 인입되는 카카오톡 대화를 모니터링하고 잘못 분배된 주문을 수동으로 복구하는 화면입니다.

### 1.1. 뱃지(Badge) 렌더링 로직
`chat_logs` 테이블의 조회 정보에 따라 다음과 같은 상태 UI 뱃지를 생성합니다.
*   `연동 대기`: 매칭된 상품이 있지만 `is_processed=false` 인 경우
*   `재고 과초`: 매칭된 상품의 잔여 수량이 부족한 경우 (현재 `UNKNOWN` 상태로 분류)
*   `상품미등록`: 매칭된 상품을 DB에서 아예 찾지 못한 경우 (빨간색 뱃지)
*   일반 텍스트(`X` 또는 `-`): `픽업고지`, `문의`, `기타` 일 때는 오역 방지를 위해 상품명과 수량을 블라인드 처리하고 뱃지 로직을 아예 스킵합니다.

### 1.2. 수동 복구 / 일괄 상태 변경 (Bulk Actions)
*   **1:1 매핑 규칙 강제:** `상품명 일괄선택` 또는 `변경` 명령을 내리면, UI는 시스템 API와 동일하게 **무조건 새로 발급된 독립된 `orders` 행과 하나의 `order_items` 행을 생성**합니다.
    (기존에는 동일 날짜/닉네임 병합(Merge) 로직이 존재했으나, 주문관리 테이블 파편화 문제로 삭제되었습니다.)
*   동기화 성공 시 `is_processed = true` 로 변경되어 대시보드 표 내의 테두리와 배지 색상이 녹색 톤으로 변합니다.

## 2. 특정일 배송/주문 관리 (Matrix Orders - `/orders/page.tsx`)
원하는 상품과 날짜를 지정하면 해당 기준에 부합하는 `order_items` 및 `orders`를 Cross-Join 하여 그리드 행태로 렌더링합니다.

## 3. 상품 관리 (Products - `/products/page.tsx`)

### 3.1. 상품 카드 모델링
*   **재고(발주량) 동기화 원리:** 카탈로그에 '발주량'을 입력하면, 실시간으로 해당 상품 관련 `order_items` 합계 수량을 빼어 **잔여 수량**을 UI에 업데이트합니다.
*   입력 칸은 스핀 버튼을 숨기기 위해 `type="text"` 및 `inputMode="numeric"` 로 구성되었으며 자바스크립트 정규표현식(`.replace(/[^0-9]/g, '')`)을 사용해 영문자 및 콤마 등 비정상 입력 에러를 원천 봉쇄합니다.
*   잔여 수량이 0개가 될 경우, 상품명 옆에 붉은색 `[🚨마감]` 스티커가 붙습니다.

### 3.2. 상품 타 매장 불러오기 (Clone)
가맹 본부 또는 타 직영점의 상품 리스트(`is_visible=true`) 목록을 조회해 현재 내 매장 ID 하위로 복제해주는 기능입니다. 같은 상품이더라도 날짜가 다를 경우 상이한 Item ID로 등록됩니다.
