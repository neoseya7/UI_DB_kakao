# [팝업 설정] 기능 구현 계획서

관리자 매장 설정의 7번째 항목의 이름을 **[7. 팝업 설정]**으로 개편하여 기존 기능과 신규 알람 기능을 통합합니다.
모든 경고 팝업은 사용자가 명시적으로 [확인] 버튼을 누르기 전까지 화면에 계속 떠 있도록(Persistent Popup) 개발합니다.

## User Review Required

> [!IMPORTANT]
> 수정된 사항은 다음과 같습니다.
> 1. 기존의 발주 시간 안내는 그대로 유지 (7번 항목 내부에 통합)
> 2. 상품 재고 0 달성 시 알림은 10분 주기가 아니라 실시간(Realtime)으로 즉시 발생
> 3. 대화 수집 지연 알림은 10분 단위로 체크
> 4. 모든 팝업은 [확인] 버튼을 클릭해야만 닫힘
> 
> 이 방식이 사장님께서 의도하신 바와 정확히 일치하는지 리뷰 부탁드립니다. (일치한다면 바로 작업을 시작하겠습니다!)

## Proposed Changes

### 1. Database (Supabase)
- **`store_settings` 테이블 컬럼 추가**
  - 기존 설정 테이블에 여러 팝업 설정을 담을 수 있도록 `popup_settings` (JSONB) 컬럼을 추가합니다.
  - 기본값으로 `{"deadline_enabled": false, "chat_enabled": false, "chat_threshold_min": 30}`를 가집니다.
  - (기존 발주시간 관련 컬럼인 `order_alert_enabled`, `alert_minutes_before`는 손대지 않고 기존대로 유지합니다.)

### 2. Frontend Settings UI (`src/app/(dashboard)/settings/page.tsx`)
- 매장 설정 7번 항목의 타이틀을 기존 "[7. 발주시간 안내 팝업 설정]"에서 **"[7. 팝업 설정]"**으로 확장 변경합니다.
- 변경된 7번 섹션 내부의 구성:
  - (기존) 발주 마감 알림 토글 및 시간 설정
  - **(신규) 상품마감 알림 여부 (토글)**
  - **(신규) 오늘의대화 업데이트 알림 여부 (토글)**
  - **(신규) 오늘의대화 알림 기준 시간 (숫자 입력란, 기본값 30분)**

### 3. Backend API (`src/app/api/admin/check-alerts/route.ts`)
- 프론트엔드가 요구할 때 팝업 상태를 체크해주기 위한 API를 생성합니다.
- 상품의 재고나 최근 대화의 생성 시간을 반환하여, 불필요한 클라이언트 로직을 최소화합니다.

### 4. Global Alert Component (`src/components/PopupManager.tsx`)
앱 전반에 걸쳐 띄울 숨김 컴포넌트(`PopupManager`)를 추가합니다.

- **규칙 A: 강제 확인형 팝업**
  팝업이 뜨면 단순 토스트(알림창)가 아니라, 관리자가 **[확인]** 버튼을 누르지 않으면 절대 안 닫히는 모달창(AlertDialog) 형태로 제작합니다.
- **규칙 B: 상품 마감 "즉시" 알림 (Realtime)**
  - Supabase의 Realtime WebSockets 기능을 사용하여 DB의 `order_items` 테이블 변경(주문 시 재고 차감)을 실시간 감지합니다.
  - 주문이 들어온 직후, 남아있는 재고를 즉시 검사하여 재고가 0이 된 상품을 발견하면 `[OOOO 상품 마감되었습니다.]` 팝업을 즉시 띄웁니다.
- **규칙 C: 오늘의 대화 수집 딜레이 알림 (10 Interval)**
  - `setInterval`을 이용하여 10분(600,000ms)마다 백그라운드에서 최근 주문의 수집된 시간표(`created_at`)를 확인합니다.
  - 이 시간이 현재 시간 대비 **N분(기본 30분)**을 초과했다면 `[오늘의대화가 수집이 안되고 있을 수 있어요 대화를 확인해주세요]` 팝업을 띄웁니다.
